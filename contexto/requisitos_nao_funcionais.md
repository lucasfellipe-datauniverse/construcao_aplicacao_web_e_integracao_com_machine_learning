## Requisitos não funcionais do software

  Abaixo estao os requisitos nao funcionais do Projeto 1 conforme discutido com as áreas
de negócio e resultado do levantamento de requisitos.

RNF1 - Desempenho: A previsão de salário deve ser realizada em no máximo 5 segundos
após a inserção de todos os dados pelo usuário.

RNF2 - Escalabilidade: A aplicação deve ser capaz de suportar um aumento no número
de usuários e consultas sem degradação significativa no desempenho.

RNF3 - Segurança: A aplicação deve garantir a segurança dos dados do usuário,
implementando medidas de proteção contra vulnerabilidades comuns da web.

RNF4 - Compatibilidade: A aplicação deve ser compatível com os principais navegadores
web, como Chrome, Firefox, Safari e Edge.

RNF5 - Manutenibilidade: O código da aplicação deve ser bem documentado e
estruturado para facilitar atualizações e manutenções futuras.
